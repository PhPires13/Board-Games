 \hypertarget{md_README_autotoc_md3}{}\doxysection{\texorpdfstring{Descrição}{Descrição}}\label{md_README_autotoc_md3}

\begin{DoxyItemize}
\item Esse projeto é uma interface de jogos de tabuleiro via terminal, utilizando Paradigma Orientado a Objetos com C++ 11.

\begin{DoxyItemize}
\item \#\#\# 1 Jogo da Velha
\item \#\#\# 2 Símbolo customizado do jogador
\begin{DoxyItemize}
\item Ao criar um jogador é possível que este escolha um símbolo para representá-\/lo no tabuleiro.
\item Com isso também é necessário tratar quais os símbolos válidos, e caso dois jogadores estejam com mesmo símbolo, mudar para os padrões.
\end{DoxyItemize}

\begin{DoxyItemize}
\item É possível escolher as proporções do tabuleiro, desde que se obedeça às regras de cada jogo, caso contrário é usado o tamanho padrão.
\end{DoxyItemize}

\begin{DoxyItemize}
\item Para melhor legibilidade, foram utilizadas cores no menu que facilitam a interpretação do usuário.

\end{DoxyItemize}
\item \#\#\# 5 Editar Jogador
\begin{DoxyItemize}
\item Foi implementada a funcionalidade de editar um jogador, podendo alterar o nome ou o símbolo, e até remover o símbolo.
\end{DoxyItemize}

\begin{DoxyItemize}
\item Criar executáveis para ambos os sistemas operacionais.
\item Usar códigos que funcionem corretamente em todos os sistemas.
\item Caso não existisse código comum, identificar o sistema e roda suas equivalências, como\+: o {\ttfamily clear} e o {\ttfamily cls} ou o {\ttfamily rm -\/f} e o {\ttfamily del /Q /F} no Makefile e outros.
\end{DoxyItemize}
\item \#\#\# 7 Estatística de Cobertura de testes

 \hypertarget{md_README_autotoc_md15}{}\doxysection{\texorpdfstring{Funcionamento}{Funcionamento}}\label{md_README_autotoc_md15}

\begin{DoxyItemize}
\item O programa roda em loop em cima da funcionalidade do {\ttfamily manager\+::menu()} a qual chama respectivas funções para os comandos, cuidando também do seu tratamento de exceções.
\item Essas funções por sua vez, tratam uma parte dos dados para encaminhar para o correto procedimento.
\item {\ttfamily CRUD \doxylink{classPlayer}{Player}}\+: após tratamento descrito acima, Sào chamas as funções da classe {\ttfamily \doxylink{classPlayer}{Player}} para completar o procedimento.
\item {\ttfamily Executar Partida}\+: após tratamento descrito acima, é escolhido qual jogo será executado, então armazenado em um ponteiro da classe genérica para o correto funcionamento do {\itshape  Dynamic Dispatch } é chamada então a função {\ttfamily play\+Game()}, e após o resultado são atualizadas as estatísticas dos jogadores.
\end{DoxyItemize}
